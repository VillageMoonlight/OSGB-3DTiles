### Bug 修复：根节点合并逻辑无效

**现象**：130 块 OSGB 转换后，Cesium 预览时所有块同时加载，出现"一块块往外蹦"的现象，加载极慢。

**根因分析**（通过查看实际生成的 `tileset.json`）：
1. `autoComputeMergeLevel` 目标阈值 100 太大：130 块只触发 1 层四叉树合并，效果极弱
2. 四叉树内部节点 `refine: ADD`：Cesium 将所有子节点同时加载，等于没有 LOD
3. `geometricError` 梯度太平缓（仅 3 倍），Cesium 判断很早就要展开所有子节点

**修复（`src/main.cpp`）**：

| 修改点 | 旧值 | 新值 |
|---|---|---|
| `autoComputeMergeLevel` 目标叶节点数 | 100 | 16（4×4 象限） |
| 四叉树内部节点 `refine` | `ADD` | `REPLACE` |
| 内部节点 `geometricError` 系数 | 3× | 6× |
| 全局根节点 `refine` | `ADD` | `REPLACE` |
| 新增 `parseBlockCoord` 失败 fallback | 无 | 按索引均分 4 组，避免无限递归 |

**验证结果（Python 模拟）**：
- 132 块 → 自动计算合并层级 = 2
- 四叉树叶节点数 ≈ 16 组（每组约 8 块）
- 全部 `refine=REPLACE` ✅，`geometricError` 单调递减（3218 → ~600 → 125）✅
